这份名为**《言灵炼金师 (Voxel Alchemist)》**的游戏开发比赛规划书，总结了我们探讨出的所有核心创意与技术路径。它规避了“AI实时写代码”的高风险，转而拥抱“物理参数化模拟”的高自由度。

---

### 一、 核心玩法 (Gameplay)

**1. 交互模式：语义驱动（Incantation）**
玩家不点击技能图标，而是通过**按住语音键实时描述**。例如：“喷射出蓝色的、粘稠的强酸液体！”或“在我脚下生成极其坚硬的浮空石块”。

**2. 战斗逻辑：涌现式物理博弈**
游戏没有预设的“火克水”规则。所有魔法都是由**2D动态体素（落砂物理）**构成的。
*   **物理博弈：** 攻击是否有效取决于物质的物理属性（如：高密度的铁球能砸碎低硬度的木墙；高传染性的毒气会绕过盾牌扩散）。
*   **环境互动：** 玩家创造的物质会永久改变地形。岩浆融化地表，粘液减速敌人，冰块封死洞口。

**3. 限制机制：语义疲劳度（Semantic Fatigue）**
为了防止玩家复读“最强咒语”（如“核弹爆炸”）：
*   **词汇耐久：** 每一个核心词汇（如“火”、“大”、“爆炸”）都有隐含的耐久度。
*   **效能衰减：** 连续使用同一个词，该词驱动的物理参数会不断减弱。
*   **倒逼创新：** 迫使玩家寻找同义词或新组合（如用“炽热的流体力”替代“喷火”），从而体现“炼金师”的学识和临场应变。

**4. 语言增益：言灵共鸣（Incantation Resonance）**
玩家可以用很多方式说出同一个法术。
生活化, 口语化, 甚至现代物象化的表达, 仍然可以被解析并施放。
但如果表达方式更贴合世界观, 更像这个世界里真正会流通的咒辞, 则系统会给予更高的言灵共鸣与魔力增益。
这意味着：
*   **可用性优先：** 先尽量让玩家“说什么都能放出来”。
*   **世界观奖励：** 更半文半白, 更文学, 更仪式化, 或更符合本世界术法语感的表达, 效果略强。
*   **不是单一文风奖励：** 生活化, 口语化, 半文半白, 文学描写都应存在分布, 差别主要体现在共鸣强弱, 而不是能不能施法。

### 二、 物质模拟系统 (Physical Parameter System)

AI 不直接生成技能代码，而是将玩家的语音翻译为以下 **6 个底层物理原子参数**：

1.  **物态 (State)：** 固态、液态、气态（决定移动逻辑：下落、流动或升腾）。
2.  **颜色 (Color)：** RGB/HEX值（直接渲染视觉，不影响逻辑，但增强反馈）。
3.  **硬度 (Hardness)：** 抵抗破坏的能力（决定是否会被其他体素替换）。
4.  **密度 (Density)：** 质量与重力响应（决定沉浮，如油浮于水，金沉于底）。
5.  **粘稠度 (Viscosity)：** 流动速度与附着力（决定是像水一样散开，还是像蜂蜜一样挂在墙上）。
6.  **传染速度 (Contagion)：** 转化相邻体素的能力（模拟燃烧、腐蚀、生长、封冻的关键）。

### 三、 技术架构 (Technical Stack)

**1. AI 管线 (Semantic Pipeline)：**
*   **STT (语音转文字)：** 使用 **Faster-Whisper**。采用流式输入，在玩家说话时实时展示单词，减少心理延迟。
*   **LLM (语义解析)：** 使用 **Groq API** (Llama-3/Mixtral) 或本地 **Qwen-0.5B**。
    *   **任务：** 充当“参数翻译官”，将模糊的描述映射为上述 6 个参数的 JSON 格式。
*   **同步：** Python 后端处理 AI，通过 **Local Socket (WebSocket)** 与游戏引擎实时通信。

**2. 渲染与物理 (Voxel Engine)：**
*   **核心技术：** **2D GPU Compute Shader** (落砂模拟算法)。
*   **碎裂系统：** 采用 *Noita* 式的连通域检测，将断裂的静态像素实时转化为 2D 刚体（Rigidbody）。
*   元胞自动机尝式的物理尝试能否并行处理
  
采用python调用模型+godot通信.

### 四、 延迟补偿与视觉反馈 (Visual Juice)

为了让“AI思考时间”不显得尴尬，采用**“渐进式塑造”**策略：
1.  **单词识别：** 当玩家说出“红色”，手心中的原始魔力团立刻变红。
2.  **属性注入：** 说出“沉重”，魔力团下沉并让角色手部下沉（增加动画细节）。
3.  **最终成型：** 语音结束的一瞬间，所有物理参数合并下发，魔法喷涌。
*这种设计让延迟变成了魔法“吟唱”和“蓄力”的过程，反而增强了沉浸感。*
